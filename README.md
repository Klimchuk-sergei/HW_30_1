# Проект LMS-платформы на Django REST Framework

Это учебный проект, созданный в рамках курса по веб-разработке. Цель проекта — разработать API для простой LMS (Learning Management System) платформы, предоставляющей информацию о курсах и уроках.

## Основные технологии

*   **Python 3.10+**
*   **Django 5.x**
*   **Django REST Framework (DRF)**
*   **PostgreSQL** (для хранения данных)
*   **pip + venv** (для управления зависимостями)

---

## Структура проекта

Проект состоит из нескольких ключевых приложений:

*   **`config/`**: Основная конфигурация Django-проекта.
*   **`users/`**: Приложение для управления пользователями. Содержит кастомную модель `User`.
*   **`course/`**: Приложение для управления курсами учебного проекта. Содержит модели `Course` (Курс).
*   **`lesson/`**: Приложение для управления учебными материалами. Содержит модели `Lesson` (Урок).

---

## Модели данных

### 1. Пользователь (`users.User`)
Кастомная модель пользователя, унаследованная от `AbstractUser`.
*   **`email`**: Основное поле для авторизации (уникальное).
*   **`phone`**: Номер телефона (необязательное).
*   **`city`**: Город (необязательное).
*   **`avatar`**: Аватар пользователя (необязательное).

### 2. Курс (`course`)
*   **`title`**: Название курса.
*   **`course_image`**: Изображение-превью для курса.
*   **`description`**: Описание курса.

### 3. Урок (`lesson`)
*   **`title`**: Название урока.
*   **`description`**: Описание урока.
*   **`lesson_imag`**: Изображение-превью для урока.
*   **`video_file`**: Ссылка на видео-материал.
*   **`course`**: Связь с моделью `Course` (ForeignKey). Один курс может содержать много уроков.

---

## API Endpoints (Эндпоинты)

Для взаимодействия с данными реализованы следующие эндпоинты:

### Курсы (`/course/`) - реализованы через `Viewsets`
*   `GET /course/` - Получение списка всех курсов.
*   `POST /course/` - Создание нового курса.
*   `GET /course/{id}/` - Получение информации о конкретном курсе.
*   `PUT /course/{id}/` - Полное обновление информации о курсе.
*   `PATCH /course/{id}/` - Частичное обновление информации о курсе.
*   `DELETE /course/{id}/` - Удаление курса.

### Уроки (`/lesson/`) - реализованы через `Generic APIView`
*   `GET /lesson/` - Получение списка всех уроков.
*   `POST /lesson/` - Создание нового урока.
*   `GET /lesson/{id}/` - Получение информации о конкретном уроке.
*   `PUT /lesson/{id}/` - Обновление информации об уроке.
*   `DELETE /lesson/{id}/` - Удаление урока.

## Установка и запуск проекта

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/Klimchuk-sergei/HW_30_1.git
    cd HW_30_1
    ```

2.  **Создайте и активируйте виртуальное окружение:**
    ```bash
    python -m venv venv
    # Для Windows
    .\venv\Scripts\activate
    # Для macOS/Linux
    source venv/bin/activate
    ```

3.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Настройте переменные окружения:**
    *   Скопируйте файл `.env.example` в `.env`.
    *   Заполните `.env` вашими данными для подключения к базе данных PostgreSQL.

5.  **Примените миграции:**
    ```bash
    python manage.py migrate
    ```

6.  **Создайте суперпользователя (опционально):**
    ```bash
    python manage.py createsuperuser
    ```

7.  **Запустите сервер для разработки:**
    ```bash
    python manage.py runserver
    ```

Проект будет доступен по адресу `http://127.0.0.1:8000/`.
